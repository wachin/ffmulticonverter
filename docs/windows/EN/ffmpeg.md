# FFmpeg on Windows 10: convert videos and audios from CMD / PowerShell (FULL guide)

This guide is intended for someone who is just learning console.. Everything you will see works the same in **PowerShell** and almost the same in **CMD** (with small differences in autocomplete).

---

## 1) Before starting: 3 key ideas

### ✅ A) Open a console in the folder where your files are (the easiest)

**Method 1 (Explorer):**

1. Open the folder where your video/audio is.
2. In the address bar (above, where the path appears), type: `powershell` and Enter.
   PowerShell will open **already located** in that folder.

**Method 2:**

* Shift + right click inside the folder (in an empty space) -> **"Open PowerShell window here"** (depends on your Windows).

### ✅ B) Always use quotes in names (because of spaces)

Example:

```powershell
ffmpeg -i "Mi video con espacios.avi" "Salida.mp4"
```

### ✅ C) Verify that FFmpeg works

```powershell
ffmpeg -version
```

---

## 2) Autocomplete with TAB (so you do not type everything by hand)

### In PowerShell (recommended)

* You type the beginning of the name and press **TAB** and it completes.
* If there are several matching files, each TAB changes between options.

Example:

1. Type:

```powershell
ffmpeg -i "
```

2. Without closing quotes, type part of the name:

```powershell
ffmpeg -i "cli
```

3. Press **TAB** -> PowerShell completes the rest of the name.

✅ About what you say about "automatically closing the quote":

* PowerShell usually **handles quotes well** and autocomplete, but the exact behavior can vary depending on configuration/version.
* The important thing: **if the name has spaces, you leave the quotes** and use TAB to complete inside.

**Super useful tip:** you can also drag the file from Explorer into the console and the path is pasted (usually already with quotes).

### In CMD

CMD also has TAB, but it is simpler:

* It completes names inside the current folder.
* It does not always place quotes automatically: that is why you put them.

---

## 3) Basic concept: structure of an FFmpeg command

The typical form is:

```powershell
ffmpeg -i "ENTRADA.ext" "SALIDA.ext"
```

* `-i` means **input**
* Then comes the output file with the format you want.

---

# PART 1 (EASY): conversions WITHOUT learning codecs

Here you do not put strange parameters. FFmpeg decides for you what is most reasonable according to the output container.

> Important: ".mp4", ".mp3", ".wav", etc. are **formats/containers** or **file formats**. Inside the video there are **codecs** (how it is compressed). In this part you do NOT worry about that.

---

## A) Video -> MP4 (the most common)

AVI / MKV / MOV -> MP4:

```powershell
ffmpeg -i "clip0001.avi" "clip0001.mp4"
```

## B) Video -> audio (extract mp3)

MP4/AVI/MKV -> MP3:

```powershell
ffmpeg -i "Mi video.mp4" "Mi audio.mp3"
```

## C) Audio -> MP3

WAV / M4A / WMA / AAC -> MP3:

```powershell
ffmpeg -i "Audio original.wav" "Audio.mp3"
```

## D) Audio -> WAV (for editing, lossless)

MP3 / M4A -> WAV:

```powershell
ffmpeg -i "Cancion.mp3" "Cancion.wav"
```

## E) Video -> GIF (for memes or demonstrations)

```powershell
ffmpeg -i "Video.mp4" "Video.gif"
```

(Note: GIF usually weighs a lot and loses quality; in the advanced part you will see better options.)

---

## What values does FFmpeg use "by default" if you do NOT put parameters?

FFmpeg does something called **"automatic selection"**:

* It chooses a **container** by the extension (for example `.mp4`).
* Inside that container, it chooses **compatible** and "reasonable" **codecs**.

But beware: the "defaults" **are not one universal rule**. They depend on:

* The output format (mp4, mkv, mp3...)
* What codecs your FFmpeg has compiled
* The version

Even so, in practice:

* For **MP4**, it normally tries something compatible (many times H.264 for video + AAC for audio if it re-encodes).
* For **MP3**, it uses an available MP3 encoder (today it is usually `libmp3lame` if it is included).

✅ **How to see exactly what FFmpeg did**
When you run the command, FFmpeg prints lines like:

* `Stream #0:0` (video) -> what codec goes in and what comes out
* `Stream #0:1` (audio) -> same

And to confirm later:

```powershell
ffprobe -hide_banner "clip0001.mp4"
```

---

## "Mass" conversions (many files) WITHOUT learning codecs

### PowerShell: convert all AVI to MP4

In the folder where the AVI files are:

```powershell
Get-ChildItem *.avi | ForEach-Object {
  ffmpeg -i "$($_.Name)" "$($_.BaseName).mp4"
}
```

### Convert all WAV to MP3

```powershell
Get-ChildItem *.wav | ForEach-Object {
  ffmpeg -i "$($_.Name)" "$($_.BaseName).mp3"
}
```

---

# PART 2 (ADVANCED): full control with codecs (better quality / size)

Here you tell FFmpeg exactly how to compress.

## A) AVI -> MP4 (H.264 + AAC) recommended

This is the classic command (like the one you put):

```powershell
ffmpeg -i "Mi video.avi" -c:v libx264 -preset medium -crf 23 -c:a aac -b:a 192k "Mi video.mp4"
```

### What does each thing mean?

* `-c:v libx264` -> **H.264** video codec (very compatible)
* `-preset medium` -> speed vs compression (ultrafast, veryfast, faster, fast, medium, slow, slower, veryslow)
* `-crf 23` -> quality (lower number = more quality and more weight)

  * 18-20: high quality
  * 21-23: good balance
  * 24-28: lighter but less quality
* `-c:a aac` -> AAC audio codec (very compatible in MP4)
* `-b:a 192k` -> audio bitrate (128k normal, 192k better, 256k high)

---

## B) "Same video, same codec": copy without recompressing (fast)

If your file is already well encoded and you only want to change the container (for example MKV -> MP4):

```powershell
ffmpeg -i "Video.mkv" -c copy "Video.mp4"
```

✅ This is **instantaneous** and does not lose quality.
⚠️ But it only works if the codecs inside are compatible with the destination container.

---

## C) Lighter MP4 video (H.265 / HEVC) (less compatible)

If you want smaller files (but some old devices fail):

```powershell
ffmpeg -i "Mi video.mp4" -c:v libx265 -preset medium -crf 28 -c:a aac -b:a 192k "Mi video_h265.mp4"
```

---

## D) Extract audio in high quality (lossless)

If you want the audio exactly as it is (if the container allows it):

```powershell
ffmpeg -i "Mi video.mp4" -vn -c:a copy "Mi audio.aac"
```

* `-vn` = "no video"
* `-c:a copy` = copies the audio without recompressing

If you want WAV (for editing):

```powershell
ffmpeg -i "Mi video.mp4" -vn "Mi audio.wav"
```

---

## E) Cut a video (without editor) — useful for tutorials

Cut from second 30 for 10 seconds:

```powershell
ffmpeg -ss 00:00:30 -i "Video.mp4" -t 00:00:10 "Recorte.mp4"
```

(Perfect cuts without recompressing are another level, but this is very useful.)

---

## F) Change resolution (e.g. 1080p -> 720p)

```powershell
ffmpeg -i "Video.mp4" -vf scale=1280:720 -c:v libx264 -preset medium -crf 23 -c:a aac -b:a 192k "Video_720p.mp4"
```

---

## G) Change FPS (for example to avoid stutters)

```powershell
ffmpeg -i "Video.mp4" -r 30 -c:v libx264 -preset medium -crf 23 -c:a aac -b:a 192k "Video_30fps.mp4"
```

---

## H) Convert audio to common formats (controlling quality)

### MP3 (good quality)

```powershell
ffmpeg -i "Audio.wav" -c:a libmp3lame -b:a 192k "Audio.mp3"
```

### AAC (for cell phones, MP4/M4A)

```powershell
ffmpeg -i "Audio.wav" -c:a aac -b:a 192k "Audio.m4a"
```

### OGG (free, used in free software)

```powershell
ffmpeg -i "Audio.wav" -c:a libvorbis -q:a 5 "Audio.ogg"
```

---

# 4) Mini "cheat sheet" of most used formats

**Video (container):** mp4, mkv, mov, avi, webm
**Common video codecs:** h264 (libx264), h265 (libx265), vp9 (libvpx-vp9), av1
**Audio (formats):** mp3, wav, m4a, aac, ogg, flac
**Common audio codecs:** aac, mp3 (libmp3lame), opus, vorbis, flac

---

# 5) Tips so you do not make mistakes

✅ **Always use quotes**:

```powershell
ffmpeg -i "Clase 1 - Video final.avi" "Clase 1 - Video final.mp4"
```

✅ **If you are in the wrong folder**, go back to the file's folder and open PowerShell there.

✅ **If you want to see what a file contains** (codecs and everything):

```powershell
ffprobe -hide_banner "Mi video.mp4"
```
